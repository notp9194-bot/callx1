## 🔴 1. KILLED CALL FIXES (Critical)

### Fix A — onTaskRemoved() — App kill hone par Firebase cleanup
**File:** `CallForegroundService.java`, `GroupCallForegroundService.java`

**Pehle kya hota tha:**
- App kill hoti thi → Firebase mein call "active" rehti thi → Remote party kabhi nahi jaanta tha call drop hua
- `CallForegroundService` START_STICKY tha → OS restart karta tha service → Zombie notification aati thi

**Ab kya hoga:**
```java
@Override
public void onTaskRemoved(Intent rootIntent) {
    // App kill → Firebase "ended" likho → Remote party ka CallActivity detect karega → Auto dismiss
    FirebaseUtils.db().getReference("activeCalls").child(callId)
        .child("status").setValue("ended");
    stopSelf();
}
```

### Fix B — Firebase onDisconnect() — Network cut par auto-cleanup
**File:** `CallActivity.java`

**Pehle kya hota tha:**
- Device ka internet cut hota tha → Firebase node kabhi clean nahi hota tha

**Ab kya hoga:**
```java
// callRef set hone ke baad:
callRef.child("status").onDisconnect().setValue("ended");
// endCall() pe ye cancel ho jaata hai taaki duplicate na ho
callRef.child("status").onDisconnect().cancel();
```

### Fix D — Camera Pause/Resume in Background
**File:** `CallActivity.java` → `onStop()` / `onStart()`

```java
@Override protected void onStop() {
    super.onStop();
    // Camera band karo jab app background mein jaye
    // (Battery save, Android 12+ green dot hide)
    if (isVideo && videoCapturer != null && capturerRunning && !isInPictureInPictureMode())
        videoCapturer.stopCapture();
}

@Override protected void onStart() {
    super.onStart();
    // Camera resume karo jab app foreground mein aaye
    if (isVideo && videoCapturer != null && !capturerRunning && camOn)
        startCapture();
}
```
